# 範例專案說明 (Example Project Guide)

## 專案概覽

本文件說明如何使用 **I&C Project Management** 模板來管理實際專案。我們將透過一個虛構的「企業網站改版專案」作為範例，展示如何套用此模板到真實情境。

---

## 範例專案背景

**專案名稱**: 企業網站改版專案  
**專案代號**: WEB-2025-001  
**專案經理**: 王小明  
**專案期間**: 2025-01-01 ~ 2025-06-30 (6個月)  
**專案預算**: NT$ 2,000,000  
**團隊規模**: 5人

### 專案目標

1. 重新設計企業官網，提升使用者體驗
2. 實現響應式設計，支援行動裝置
3. 整合 CMS 系統，方便內容管理
4. 優化 SEO，提升搜尋引擎排名
5. 提升網站載入速度 50%

### 專案範圍

**包含**:
- 首頁、產品頁、關於我們、聯絡我們、部落格
- 後台管理系統
- 會員系統
- 購物車功能

**不包含**:
- 金流串接 (第二期)
- 物流系統 (第二期)
- APP 開發

## 如何套用模板

### 步驟 1: 建立 Teams 團隊

1. 開啟 Microsoft Teams
2. 建立新團隊：「企業網站改版專案」
3. 邀請團隊成員：
   - 王小明 (專案經理)
   - 李設計 (UI/UX 設計師)
   - 張前端 (前端工程師)
   - 陳後端 (後端工程師)
   - 林測試 (QA 測試工程師)

4. 建立頻道：
   - 📋 一般 (預設)
   - 🚀 專案啟動
   - 🎨 設計討論
   - 💻 開發討論
   - 🧪 測試與 QA
   - 📈 週報與檢討
   - ✅ 結案報告

### 步驟 2: 建立 Planner 看板

1. 在 Teams 中新增 Planner 分頁
2. 建立計畫：「企業網站改版專案 - Master Plan」
3. 建立 Buckets：
   - 🔵 待辦 (To Do)
   - 🟡 進行中 (In Progress)
   - 🟢 審核中 (Review)
   - 🟣 已完成 (Done)
   - 🔴 已封存 (Archived)

### 步驟 3: 建立 OneNote 筆記本

1. 在 Teams 中新增 OneNote 分頁
2. 建立筆記本：「企業網站改版專案 Notebook」
3. 建立區段：
   - 📌 專案章程
   - 🎯 專案目標與範圍
   - 👥 利害關係人
   - 📅 會議紀錄
   - 🎨 設計文件
   - 💡 技術文件
   - 📚 知識庫
   - ✅ 結案文件

## 範例數據解讀

### Dashboard 數據範例

假設專案進行到第 10 週 (2025-03-10)，Dashboard 顯示：

**專案總覽**:
- 總任務數: 45
- 已完成: 18 (40%)
- 進行中: 12 (27%)
- 待辦: 15 (33%)
- 逾期任務: 2 (4%)

**本週完成任務**: 5
**本週新增任務**: 3
**整體進度**: 40%

### 如何解讀

1. **進度正常**: 40% 進度符合預期 (10週/24週 ≈ 42%)
2. **逾期任務需關注**: 2 個逾期任務需要立即處理
3. **工作量穩定**: 本週完成 5 個任務，新增 3 個任務，工作量穩定
4. **進行中任務較多**: 12 個進行中任務，需確認是否有阻礙

### 甘特圖解讀

- **藍色區塊**: 任務預定期間
- **深藍色區塊**: 已完成進度
- **紅色虛線**: 今日日期
- **菱形標記**: 里程碑

**觀察重點**:
- 是否有任務落後進度？
- 關鍵路徑上的任務是否準時？
- 資源分配是否合理？

## 如何套用到實際專案

### 1. 複製模板

將以下檔案複製到您的專案資料夾：
- `Dashboard.xlsx`
- `Task_Tracker.xlsx`
- `Weekly_Report_Template.xlsx`

### 2. 自訂專案資訊

在 `Dashboard.xlsx` 的「專案總覽」Sheet 中：
- 修改專案名稱
- 修改專案經理
- 修改開始/結束日期
- 修改專案狀態

### 3. 建立任務清單

在「任務清單」Sheet 中：
- 刪除範例任務
- 新增您的實際任務
- 設定負責人、優先級、截止日期

### 4. 設定負責人清單

在「任務清單」Sheet 的資料驗證中：
- 修改負責人下拉選單
- 新增您的團隊成員名稱

### 5. 調整甘特圖日期範圍

在「甘特圖」Sheet 中：
- 修改起始日期 (G1)
- 調整顯示天數

### 6. 自訂統計分析

在「統計分析」Sheet 中：
- 更新負責人清單
- 調整圖表樣式
- 新增自訂統計項目

## 常見情境處理

### 情境 1: 任務逾期

**問題**: 任務 #008「技術架構設計」逾期 3 天

**處理步驟**:
1. 在週報中標記為逾期任務
2. 與負責人 (王五) 溝通了解原因
3. 評估影響範圍
4. 調整後續任務時程
5. 記錄於風險議題清單

### 情境 2: 新增緊急任務

**問題**: 客戶臨時要求新增「多語系功能」

**處理步驟**:
1. 評估工作量與影響
2. 在 Planner 新增任務
3. 設定為 🔴高優先級
4. 調整其他任務優先級
5. 更新專案範圍文件
6. 通知利害關係人

### 情境 3: 團隊成員請假

**問題**: 張前端請病假 1 週

**處理步驟**:
1. 檢視張前端的任務清單
2. 評估哪些任務可延後
3. 將緊急任務重新分配
4. 更新 Planner 負責人
5. 通知相關成員
6. 記錄於週報

### 情境 4: 專案範圍變更

**問題**: 客戶要求取消「購物車功能」

**處理步驟**:
1. 更新專案範圍文件
2. 刪除或封存相關任務
3. 重新計算專案進度
4. 調整資源分配
5. 更新專案章程
6. 發送變更通知

## 進階技巧

### 1. 使用 Power Automate 自動化

建立自動化流程：
- 新任務自動通知負責人
- 逾期任務每日提醒
- 週報自動發送

### 2. 整合 Power BI

建立進階儀表板：
- 即時專案進度
- 趨勢分析
- 預測分析

### 3. 使用 SharePoint

建立專案網站：
- 集中文件管理
- 版本控制
- 權限管理

### 4. 自訂 Excel 公式

進階公式範例：
```excel
// 計算專案健康度
=IF(逾期任務數>5,"不健康",IF(逾期任務數>2,"需關注","健康"))

// 預測完成日期
=開始日期+(總任務數/平均每週完成數)*7

// 資源利用率
=進行中任務數/團隊人數
```
