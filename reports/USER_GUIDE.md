## 1. 系統概覽

### 1.1 系統架構

本專案管理方案整合以下 Microsoft 365 工具：

```
┌─────────────────────────────────────────────────┐
│           Microsoft Teams (協作中樞)             │
├─────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐      │
│  │ Planner  │  │ OneNote  │  │  Excel   │      │
│  │ 任務看板  │  │ 筆記文件  │  │ 儀表板   │      │
│  └──────────┘  └──────────┘  └──────────┘      │
├─────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐                    │
│  │  To Do   │  │ Outlook  │                    │
│  │ 個人任務  │  │ 行事曆   │                    │
│  └──────────┘  └──────────┘                    │
└─────────────────────────────────────────────────┘
```

### 1.2 工具職責

| 工具        | 主要功能                  | 使用時機               |
| ----------- | ------------------------- | ---------------------- |
| **Planner** | 團隊任務看板、Kanban 管理 | 建立任務、追蹤團隊進度 |
| **To Do**   | 個人任務清單、每日規劃    | 管理個人工作、設定提醒 |
| **OneNote** | 專案文件、會議紀錄        | 記錄會議、設計文件     |
| **Excel**   | 儀表板、甘特圖、統計分析  | 進度追蹤、數據分析     |
| **Teams**   | 團隊溝通、檔案共享        | 協作討論、整合工具     |
| **Outlook** | 行事曆、郵件通知          | 會議排程、任務提醒     |

### 1.3 5 大核心流程

1. **專案啟動** - 定義目標、建立計畫
2. **任務分配** - 拆解任務、指派負責人
3. **日常追蹤** - 更新狀態、記錄進度
4. **週檢討** - 分析進度、調整計畫
5. **結案總報告** - 統計成果、經驗總結

## 3. Microsoft Planner 使用說明

### 3.1 建立任務

#### 步驟

1. 開啟 Planner 看板
2. 點擊任一 Bucket 下方的「+ 新增任務」
3. 輸入任務名稱
4. 按 Enter 快速建立，或點擊任務卡片進入詳細設定

#### 詳細設定

- **任務名稱**: 簡潔明確，例如「撰寫專案章程」
- **截止日期**: 點擊日期圖示設定
- **指派對象**: 點擊人像圖示選擇負責人
- **優先級**: 使用標籤（🔴高、🟡中、🟢低）
- **步驟**: 新增子任務檢查清單
- **附註**: 詳細說明或連結至 OneNote

### 3.2 拖曳任務改變狀態

#### Kanban 工作流程

```
🔵 待辦 → 🟡 進行中 → 🟢 審核中 → 🟣 已完成 → 🔴 已封存
```

#### 操作方式

1. 點擊任務卡片並按住
2. 拖曳至目標 Bucket
3. 放開滑鼠即可

#### 快捷鍵

- `Ctrl + ←/→`: 移動任務至左/右 Bucket
- `Enter`: 編輯任務
- `Esc`: 關閉任務詳情

### 3.3 新增步驟（子任務）

#### 為什麼使用步驟？

- 將大任務拆解為可執行的小步驟
- 追蹤任務完成進度
- 確保不遺漏重要細節

#### 操作步驟

1. 開啟任務卡片
2. 找到「步驟」區塊
3. 點擊「新增步驟」
4. 輸入步驟名稱
5. 按 Enter 繼續新增

#### 範例

**任務**: 撰寫專案章程

**步驟**:
- [ ] 定義專案目標
- [ ] 識別利害關係人
- [ ] 設定成功標準
- [ ] 確認預算與資源
- [ ] 審核與簽核

### 3.4 設定標籤與優先級

#### 標籤系統

| 標籤       | 顏色 | 用途         |
| ---------- | ---- | ------------ |
| 🔴 高優先級 | 紅色 | 緊急且重要   |
| 🟡 中優先級 | 黃色 | 重要但不緊急 |
| 🟢 低優先級 | 綠色 | 可延後處理   |
| 💼 管理任務 | 藍色 | 管理類工作   |
| 🔧 技術任務 | 紫色 | 技術類工作   |
| 📄 文件任務 | 橙色 | 文件撰寫     |

#### 設定方式

1. 開啟任務卡片
2. 點擊「標籤」
3. 勾選適用的標籤（可多選）

#### 優先級判斷矩陣

```
緊急度
  ↑
  │  🔴 高優先級  │  🟡 中優先級
  │  (立即處理)   │  (排程處理)
  │───────────────┼───────────────
  │  🟡 中優先級  │  🟢 低優先級
  │  (委派他人)   │  (有空再做)
  └────────────────────────→ 重要度
```

### 3.5 指派負責人

#### 單一負責人原則

- 每個任務應有明確的單一負責人
- 避免責任分散
- 可新增協作者但不影響主要負責人

#### 操作步驟

1. 開啟任務卡片
2. 點擊「指派」
3. 選擇負責人
4. （可選）新增其他協作者

#### 負責人會收到

- Teams 通知
- Outlook 郵件通知
- To Do 自動同步任務

### 3.6 設定截止日期

#### 最佳實踐

- **明確日期**: 使用具體日期，避免「盡快」
- **合理緩衝**: 預留 20% 緩衝時間
- **里程碑對齊**: 確保與專案里程碑一致

#### 操作步驟

1. 開啟任務卡片
2. 點擊「截止日期」
3. 選擇日期
4. （可選）設定提醒時間

#### 提醒設定

- 截止日前 3 天
- 截止日前 1 天
- 截止日當天早上 9:00

## 4. Microsoft To Do 使用說明

### 4.1 我的一天規劃

#### 概念

「我的一天」是 To Do 的核心功能，幫助您專注於今日最重要的任務。

#### 每日規劃流程

**早上 9:00（5 分鐘）**

1. 開啟 To Do
2. 點擊「我的一天」
3. 查看建議任務（系統自動推薦）
4. 從「我的任務」拖曳今日要完成的 3-5 個任務
5. 排序優先順序（最重要的放最上面）

**下班前（5 分鐘）**

1. 檢查今日完成狀況
2. 未完成任務決定：
   - 移至明日
   - 重新評估優先級
   - 或刪除（不重要）

### 4.2 設定提醒

#### 提醒類型

1. **日期提醒**: 特定日期
2. **時間提醒**: 特定時間
3. **地點提醒**: 到達特定地點時（行動裝置）
4. **重複提醒**: 每日/每週/每月

#### 操作步驟

1. 開啟任務
2. 點擊「提醒我」
3. 選擇提醒類型
4. 設定日期/時間
5. （可選）設定重複頻率

#### 範例

**任務**: 每日站立會議
- 提醒: 每日早上 9:00
- 重複: 每個工作日

**任務**: 週報撰寫
- 提醒: 每週五下午 3:00
- 重複: 每週

### 4.3 與 Planner 同步

#### 自動同步

- Planner 中指派給您的任務會自動出現在 To Do 的「已指派給我」清單
- 在 To Do 完成任務會同步至 Planner
- 雙向同步，即時更新

#### 管理同步任務

1. **在 To Do 中查看**:
   - 開啟「已指派給我」清單
   - 查看所有 Planner 任務

2. **新增至我的一天**:
   - 右鍵點擊任務
   - 選擇「新增至我的一天」

3. **設定優先級**:
   - 點擊星號標記為重要
   - 重要任務會出現在「重要」清單

## 6. Excel Dashboard 使用說明

### 6.1 任務清單更新

#### 每日更新流程

1. 開啟 `Dashboard.xlsx`
2. 切換至「任務清單」Sheet
3. 更新以下欄位：
   - **狀態**: 從下拉選單選擇
   - **進度%**: 輸入 0-100
   - **完成日期**: 任務完成時填入

#### 新增任務

1. 在最後一行下方新增一列
2. 填寫必填欄位：
   - 任務 ID（自動編號）
   - 任務名稱
   - 負責人（下拉選單）
   - 優先級（下拉選單）
   - 截止日期
   - 狀態（下拉選單）
   - 類別（下拉選單）

#### 條件格式自動提示

- **紅色粗體**: 任務已逾期
- **黃色背景**: 3 天內到期
- **綠色背景**: 已完成任務
- **藍色背景**: 進行中任務

### 6.2 甘特圖閱讀

#### 圖例說明

```
🔵 藍色: 任務期間
🔷 深藍色: 已完成進度
🔴 紅色虛線: 今日線
💎 菱形: 里程碑
```

#### 如何閱讀

1. **Y 軸**: 任務名稱
2. **X 軸**: 日期（以週為單位）
3. **橫條長度**: 任務期間
4. **深藍色部分**: 已完成的進度
5. **紅色線**: 今日位置

#### 分析重點

- **關鍵路徑**: 找出最長的任務鏈
- **資源衝突**: 同一負責人的任務是否重疊
- **進度落後**: 深藍色部分未達今日線的任務

### 6.3 週報生成

#### 自動生成步驟

1. 開啟「週報」Sheet
2. 確認週報期間自動計算正確
3. 檢查自動統計數據：
   - 本週完成任務數
   - 本週逾期任務數
   - 整體進度變化

4. 手動填寫：
   - 本週成就
   - 本週挑戰
   - 下週計畫

5. 檢視圖表：
   - 任務完成趨勢
   - 負責人工作量

6. 匯出 PDF：
   - 檔案 > 匯出 > 建立 PDF
   - 選擇「週報」Sheet
   - 儲存為 `週報_YYYYMMDD.pdf`

### 6.4 圖表解讀

#### 專案總覽儀表板

**圖表 1: 任務狀態分布（圓餅圖）**
- 顯示待辦、進行中、審核中、已完成的比例
- **健康指標**: 已完成 > 40%, 待辦 < 30%

**圖表 2: 優先級分布（環形圖）**
- 顯示高、中、低優先級任務比例
- **警示**: 高優先級 > 50% 表示資源緊張

**圖表 3: 團隊成員工作量（橫條圖）**
- 顯示各成員任務數
- **平衡檢查**: 工作量差異不應超過 2 倍

**圖表 4: 完成趨勢（折線圖）**
- 顯示每週完成任務數趨勢
- **理想曲線**: 穩定上升或維持平穩

## 7. Teams 協作說明

### 7.1 頻道使用

#### 頻道架構

| 頻道         | 用途         | 主要工具                |

#### 檔案管理最佳實踐

1. **命名規則**: `YYYYMMDD_文件名稱_版本.副檔名`
   - 範例: `20250106_專案章程_v1.0.pdf`

2. **版本控制**: Teams 自動保存版本歷史
   - 右鍵點擊檔案 > 版本歷程記錄

3. **共同編輯**: Excel 和 OneNote 支援即時協作

   - 自動同步變更

4. **權限管理**:
   - 團隊成員: 可編輯
   - 利害關係人: 僅檢視

## 10. 常見問題 FAQ

### Q1: Planner 任務沒有同步到 To Do？
**A**: 
1. 確認任務已指派給您
2. 重新整理 To Do 應用程式
3. 登出後重新登入
4. 檢查網路連線

### Q2: Excel 公式顯示錯誤？
**A**:
1. 檢查資料來源範圍是否正確
2. 確認欄位名稱沒有變更
3. 檢查是否有空白儲存格
4. 使用 F9 重新計算

### Q3: 如何刪除 Planner 任務？
**A**:
1. 開啟任務卡片
2. 點擊右上角「...」
3. 選擇「刪除」
4. 確認刪除
5. **注意**: 刪除後無法復原，建議移至「已封存」

### Q4: OneNote 無法同步？
**A**:
1. 檢查網路連線
2. 點擊「檔案」>「資訊」>「檢視同步狀態」
3. 手動同步：「檔案」>「資訊」>「立即同步」
4. 重新啟動 OneNote

### Q5: 如何匯出 Planner 資料？
**A**:
1. 開啟 Planner
2. 點擊右上角「...」
3. 選擇「匯出計畫至 Excel」
4. 下載 Excel 檔案

### Q6: Teams 會議錄影在哪裡？
**A**:
1. 錄影自動儲存至 OneDrive
2. 在 Teams 聊天中會有連結
3. 也可在「會議紀錄」頻道的「檔案」中找到

### Q7: 如何設定重複任務？
**A**:
Planner 本身不支援重複任務，解決方案：
1. 使用 To Do 建立重複任務
2. 或在 Planner 手動建立多個任務（例如：每日站立會議 1/1, 1/2, 1/3...）
3. 或使用 Power Automate 自動建立

### Q8: Excel 條件格式沒有顯示？
**A**:
1. 檢查條件格式規則是否正確
2. 確認套用範圍包含目標儲存格
3. 檢查規則優先順序（規則管理員）
4. 清除格式後重新套用

### Q9: 如何新增團隊成員？
**A**:
1. 開啟 Teams 團隊
2. 點擊團隊名稱旁的「...」
3. 選擇「新增成員」
4. 輸入郵件地址或名稱
5. 設定權限（成員或擁有者）

### Q10: 如何備份專案資料？
**A**:
1. **Planner**: 匯出至 Excel
2. **OneNote**: 匯出為 PDF（檔案 > 匯出）
3. **Excel**: 下載至本地
4. **Teams 檔案**: 在 SharePoint 中下載整個資料夾

## 11. 最佳實踐

### 11.1 任務管理

✅ **DO (建議做法)**
- 任務名稱使用動詞開頭（例如：「撰寫」、「設計」、「測試」）
- 每個任務有明確的單一負責人
- 大任務拆解為可在 1-3 天完成的小任務
- 使用步驟功能建立檢查清單
- 定期更新任務狀態和進度

❌ **DON'T (避免做法)**
- 任務名稱過於模糊（例如：「處理問題」）
- 一個任務指派多個負責人
- 任務過大無法追蹤進度
- 長時間不更新任務狀態
- 使用備註記錄重要資訊後不同步至文件

### 11.2 時間管理

✅ **DO**
- 每日早上規劃「我的一天」
- 使用番茄工作法（25 分鐘專注 + 5 分鐘休息）
- 預留 20% 緩衝時間
- 設定提醒避免遺忘
- 下班前檢視今日完成狀況

❌ **DON'T**
- 同時處理多個任務
- 截止日期設定過於緊迫
- 忽略提醒通知
- 拖延重要任務
- 過度承諾

### 11.3 溝通協作

✅ **DO**
- 使用 @提及確保訊息被看到
- 在 Teams 頻道討論保持透明
- 會議後整理行動項目
- 定期同步專案狀態
- 主動分享進度和問題

❌ **DON'T**
- 重要訊息只用私訊
- 會議沒有議程
- 會後不追蹤行動項目
- 問題隱藏不報告
- 單打獨鬥不尋求協助

### 11.4 文件管理

✅ **DO**
- 使用一致的命名規則
- 定期備份重要文件
- 使用版本控制
- 文件分類清楚
- 重要決策記錄至 OneNote

❌ **DON'T**
- 隨意命名檔案
- 只有本地備份
- 覆蓋舊版本
- 文件散落各處
- 口頭決策不記錄

### 11.5 數據分析

✅ **DO**
- 每週檢視 Dashboard
- 使用數據支持決策
- 識別趨勢和模式
- 分析根本原因
- 數據視覺化呈現

❌ **DON'T**
- 憑感覺做決策
- 忽略數據警訊
- 只看表面數字
- 數據不更新
- 過度依賴單一指標

## 12. 疑難排解

### 12.1 Planner 問題

#### 問題：無法拖曳任務
**可能原因**:
- 瀏覽器快取問題
- 權限不足
- 網路延遲

**解決方案**:
1. 清除瀏覽器快取
2. 使用 Ctrl+F5 強制重新整理
3. 確認您有編輯權限
4. 嘗試使用其他瀏覽器

#### 問題：任務消失
**可能原因**:
- 被其他人刪除
- 篩選器設定
- 同步延遲

**解決方案**:
1. 檢查「已封存」Bucket
2. 清除所有篩選器
3. 檢查活動記錄
4. 聯絡團隊成員確認

### 12.2 Excel 問題

#### 問題：公式錯誤 #REF!
**原因**: 參照的儲存格被刪除

**解決方案**:
1. 復原刪除動作（Ctrl+Z）
2. 修正公式參照
3. 使用表格名稱參照而非儲存格位址

#### 問題：圖表不更新
**原因**: 資料來源範圍固定

**解決方案**:
1. 右鍵點擊圖表 > 選取資料
2. 調整資料範圍
3. 使用動態範圍（表格或 OFFSET 函數）

### 12.3 Teams 問題

#### 問題：無法新增分頁
**可能原因**:
- 權限不足
- 應用程式未啟用

**解決方案**:
1. 確認您是團隊擁有者或成員
2. 檢查組織是否允許該應用程式
3. 聯絡 IT 管理員

#### 問題：檔案無法開啟
**可能原因**:
- 檔案損毀
- 權限問題
- 應用程式未安裝

**解決方案**:
1. 檢查檔案版本歷程記錄，還原舊版本
2. 確認您有存取權限
3. 安裝對應的 Office 應用程式

### 12.4 同步問題

#### 問題：Planner 與 To Do 不同步
**解決方案**:
1. 登出後重新登入兩個應用程式
2. 檢查網路連線
3. 等待 5-10 分鐘（同步有延遲）
4. 清除應用程式快取

#### 問題：OneNote 同步衝突
**解決方案**:
1. 檢視同步狀態
2. 手動合併衝突頁面
3. 刪除重複頁面
4. 確保只在一個裝置編輯
